# 算法和数据结构
## 算法
- 解决问题的方法的时间和空间的复杂度
- 分类
    - 穷举法
    - 均摊分析
        - 二分法
        - 归并排序
    - 贪心: 最小生成树,单源最短路
    - 动态规划: 背包问题,士兵路径
- 实现算法之前,要预估算法所需要的资源
    - 时间资源(基本操作次数)
    - 空间资源(占用内存字节数)
    - 区别: 空间可以再利用
    - 时空互换(Hash表)
- 复杂度
    - `O(1)`基本运算
    - `O(logn)`二分查找 : 例如分支
    - `O(n^1/2)`枚举约数
    - `O(n)`线性查找 : 例如单重for循环
    - `O(n^2)`朴素最近点对: 例如双重for循环
    - `O(nlogn)`
        - 归并排序
        - **快速排序的期望复杂度**
        - 基于比较排序的算法下界
    - n指n个整数,m指范围
    - 总结
        - 优秀`O(1) < O(logn) < O(n^1/2) < O(n) < O(nlogn)`
        - 可能可以优化 `O(n^2) < O(n^3) < O(2^n) < O(n!)`
- 怎么算复杂度
    - 输入输出法
    - 数循环次数
    - 均摊方法三
